Fundamentos de la automatización agéntica

Para construir una base sólida, primero debemos entender qué diferencia a un "bot" de un "agente".

RPA Tradicional

Funciona bajo el principio "Si A, entonces B". Es determinista y se rompe si la interfaz cambia un milímetro. Ideal para procesos 100% predecibles, alto volumen y cero variabilidad.

Automatización Agéntica

Funciona bajo el principio "Logra el objetivo X usando las herramientas disponibles". Es probabilística y utiliza LLMs como cerebro para navegar la incertidumbre.

Tabla Comparativa

  • Lógica: RPA = determinista | Agente = probabilístico
  • Input: RPA = datos estructurados | Agente = cualquier tipo
  • Cambios: RPA = se rompe | Agente = se adapta
  • Cerebro: RPA = tú (código) | Agente = LLM
  • Ideal para: RPA = reglas fijas | Agente = variabilidad
La magia de los agentes brilla en procesos semi-estructurados, como gestionar correos de clientes o comparar presupuestos, donde las reglas cambian constantemente.

Un agente no ejecuta instrucciones ciegas. Descompone una tarea compleja en pasos lógicos y razona antes de actuar.

Chain of Thought (CoT)

Técnica que fuerza al modelo a "pensar en voz alta" paso a paso antes de dar una respuesta final. Aumenta la precisión en tareas complejas. Ejemplo: "Primero analiza el monto, luego compara con el umbral, finalmente decide si solicitar aprobación."

Framework ReAct (Reason + Act)

El agente alterna entre razonar sobre lo que ve y ejecutar una acción. Este ciclo continúa hasta completar la tarea:

  • Thought: "Necesito saber el saldo del cliente"
  • Action: Llama a la API de CRM
  • Observation: "Saldo = $4,500"
  • Thought: "Supera el umbral, debo notificar"
  • Action: Envía mensaje por Slack

Agente Reflexivo

Antes de entregar la respuesta, el agente evalúa su propio razonamiento. Si detecta un error, lo corrige internamente.

No intentes automatizar procesos perfectos desde el día uno. Empieza con procesos que tengan variabilidad media.

Un agente sin herramientas es solo un chat. Las "herramientas" son las manos del agente: le permiten interactuar con el mundo real.

Tipos de Herramientas

  • APIs REST: Consultar datos de sistemas externos (CRM, ERP).
  • Navegadores web: Interactuar con interfaces gráficas.
  • Scripts Python: Cálculos complejos, procesamiento de datos.
  • Conectores nativos: Slack, Gmail, Excel, Salesforce.

Function Calling

Mecanismo donde el LLM decide cuándo y cómo invocar una función. El modelo no ejecuta directamente, sino que genera los parámetros necesarios y el sistema los ejecuta.

Tool Registry

Catálogo de herramientas disponibles que el agente puede consultar. El agente lee las descripciones y decide cuál es la más adecuada para cada subtarea.

UiPath Studio Web

Plataforma para diseñar las herramientas que los agentes usarán. Familiarízate con las actividades de integración: Slack, Gmail, Excel, etc.

Describe bien cada herramienta en el Tool Registry. El agente elige basándose en las descripciones, no en el código interno.

¿Cómo recuerda el agente lo que hizo hace tres pasos? Existen diferentes tipos de memoria:

Memoria a Corto Plazo

El historial del chat y el contexto inmediato. Limitada por la ventana de contexto (tokens). Permite mantener coherencia en la tarea actual.

Memoria a Largo Plazo (RAG)

Uso de bases de datos vectoriales para consultar manuales, políticas o datos históricos. El agente recupera solo la información relevante para la tarea.

Memoria Episódica

Registro de interacciones pasadas con un usuario. Permite personalizar futuras respuestas: "La semana pasada aprobaste facturas de este proveedor."

Memoria Semántica

Base de conocimiento estructurado: catálogos de productos, políticas internas, procedimientos. Se consulta mediante búsqueda semántica.

Scratchpad

Espacio de trabajo temporal donde el agente anota su razonamiento intermedio, cálculos y observaciones durante la ejecución.

El Context Window define el límite de tokens que el LLM puede "ver" a la vez. Optimiza qué información incluyes en cada llamada.

Incluso los mejores agentes necesitan supervisión. El HITL son puntos de control donde el agente se detiene y solicita validación humana antes de ejecutar acciones críticas.

¿Cuándo usar HITL?

  • Transacciones financieras que superen un umbral.
  • Acciones irreversibles (eliminar datos, enviar comunicaciones masivas).
  • Cuando la confianza del agente es baja.
  • Decisiones que afectan a clientes o regulación.

Diseño de Puntos de Control

Un buen automatizador agéntico sabe dónde colocar estos puntos. Demasiados interrumpen el flujo; muy pocos generan riesgo.

Transferencia (Hand-off)

Mecanismo para pasar la tarea entre agentes especializados o hacia un robot RPA tradicional cuando la situación lo requiere.

Define umbrales claros: "Si el monto supera $1,000, solicita aprobación por Slack antes de continuar."

No es un simple mensaje. El Agentic Prompting es darle al agente una identidad completa: quién es, qué debe lograr y qué límites tiene.

Anatomía de un Prompt Agéntico

  • Identidad (Persona): "Eres un analista de facturas experto."
  • Objetivo: "Tu tarea es revisar facturas y detectar anomalías."
  • Herramientas disponibles: Lista de tools con sus descripciones.
  • Restricciones: "Nunca apruebes montos >$5,000 sin validación."
  • Formato de salida: "Responde solo en JSON con campos: acción, razón, monto."

Técnicas Avanzadas

  • Zero-shot: Sin ejemplos previos; el modelo infiere.
  • Few-shot: Con 2-5 ejemplos para calibrar la respuesta.
  • Chain of Thought: "Piensa paso a paso antes de responder."

Salida JSON

Formato ideal para que la respuesta sea procesable por el sistema downstream. Siempre define el esquema exacto en el prompt.

System Prompt = instrucción base permanente. User Prompt = tarea específica de la sesión. Sepáralos claramente.

La coordinación de múltiples agentes especializados permite resolver problemas complejos que un solo agente no podría manejar.

Arquitecturas

  • Cadena (Chain): Agente A pasa su output a Agente B secuencialmente.
  • Paralelo: Múltiples agentes ejecutan subtareas simultáneamente.
  • Supervisor: Un agente orquestador coordina y valida el trabajo de subagentes.
  • Especialista: Cada agente tiene un dominio específico (facturación, RRHH, soporte).

Frameworks Populares

  • LangChain: Orquestación flexible con cadenas de LLMs.
  • CrewAI: Coordina agentes con roles como si fuera un equipo.
  • Microsoft AutoGen: Conversaciones entre agentes para resolver tareas.
  • Semantic Kernel: SDK de Microsoft para integrar LLMs en .NET/Python.

UiPath Orchestrator

Gobierna la ejecución y los recursos de los agentes. Permite monitorizar, reintentar y auditar cada paso.

Cada agente debe tener un rol claro y limitado. La especialización mejora la precisión y reduce las alucinaciones.

Un agente poderoso sin restricciones es un riesgo. Los guardrails son las barreras que protegen el sistema de comportamientos no deseados.

Tipos de Amenazas

  • Alucinación: El agente genera información falsa con apariencia de verdad.
  • Jailbreaking: El usuario manipula al agente para saltarse sus reglas.
  • Prompt Injection: Datos maliciosos en el input alteran el comportamiento.

Guardrails Técnicos

  • Validación de output: Verificar que la respuesta cumple el esquema JSON esperado.
  • Grounding: Anclar respuestas a fuentes de datos verificadas.
  • Rollback: Capacidad de deshacer acciones si algo sale mal.
  • Idempotencia: Las acciones pueden repetirse sin efectos adicionales.
  • Rate Limiting: Controlar la frecuencia de llamadas a APIs.

Observabilidad

Capacidad de monitorear, trazar y depurar el comportamiento del agente en producción. Incluye logs, trazas y métricas de latencia.

El Grounding es la primera defensa contra las alucinaciones. Siempre cita la fuente de los datos usados en la respuesta.

UiPath ha construido un ecosistema completo para combinar RPA tradicional con IA agéntica.

Componentes Clave

  • UiPath Studio Web: Diseño visual de herramientas y flujos agénticos.
  • UiPath Agent Builder: Creación y configuración de agentes de IA.
  • UiPath Orchestrator: Gestión centralizada de ejecución y recursos.
  • Autopilot: Asistente de IA generativa para desarrolladores y usuarios finales.

Integraciones Nativas

Conectores listos para usar con: Slack, Gmail, Microsoft Teams, Excel, Salesforce, SAP, ServiceNow y más de 300 aplicaciones empresariales.

Flujo de Trabajo Típico

  • 1. Definir el agente y su persona en Agent Builder.
  • 2. Crear/registrar las tools en Studio Web.
  • 3. Configurar los puntos HITL en Orchestrator.
  • 4. Desplegar y monitorizar con observabilidad integrada.
Autopilot puede generar código de automatización en lenguaje natural. Úsalo para acelerar el desarrollo.

Mejores Prácticas

  • Empieza pequeño: Un agente para un proceso semi-estructurado específico.
  • Itera el prompt: Trata el system prompt como código: versiona y prueba.
  • Caching: Almacena respuestas frecuentes para reducir latencia y costo.
  • Evals sistemáticas: Mide precisión y fiabilidad antes de producción.
  • Temperatura baja en producción: Reduce la aleatoriedad para mayor consistencia.

Errores Comunes

  • Over-engineering: Usar agentes para procesos que RPA maneja perfectamente.
  • Sin HITL: Dejar al agente operar sin puntos de control en acciones críticas.
  • Herramientas mal descritas: El agente elegirá la herramienta incorrecta.
  • Ignorar los tokens: Prompts infinitos consumen contexto y presupuesto.
  • Sin observabilidad: Imposible depurar fallos en producción.

¿Cuándo NO usar Agentes?

  • Proceso 100% determinista y de alto volumen.
  • Regulación estricta que exige trazabilidad exacta.
  • Latencia crítica (<100ms) que LLMs no pueden garantizar.
La combinación óptima: usa RPA para los pasos predecibles y agentes para la parte ambigua. ¡No es una competencia!

Básico Diseña tu primer Prompt Agéntico ›

Aprende a estructurar un prompt completo que convierta un LLM en un agente especializado con identidad, objetivo, herramientas y restricciones bien definidas.

📋 Tu misión
  • Define la Persona: Rol y experiencia del agente.
  • Define el Objetivo: Qué debe lograr en términos de outcome.
  • Lista 3 Herramientas con descripción de cuándo usarlas.
  • Establece 3 Restricciones claras de comportamiento.
  • Define el Formato de Salida en JSON.
✅ Criterios de éxito
  • El prompt tiene las 5 secciones completas.
  • Las restricciones son específicas y medibles.
  • El esquema JSON está bien definido con tipos de datos.
Básico Mapa de Memoria para un Agente ›

Dado un proceso de gestión de solicitudes de clientes, identifica qué tipo de memoria usar en cada situación y justifica tu elección.

📋 Escenarios a resolver
  • El agente procesa el email actual del cliente. ¿Qué tipo de memoria usa?
  • El cliente pregunta por una política de devoluciones de 50 páginas. ¿Qué tipo de memoria?
  • El cliente dice "como la última vez que llamé...". ¿Qué tipo de memoria?
  • El agente necesita recordar que ya envió una respuesta hace 5 minutos. ¿Cuál?
✅ Criterios de éxito
  • Cada escenario tiene el tipo correcto: corto plazo, RAG o episódica.
  • Justificas por qué NO usarías los otros tipos.
  • Propones cómo implementarlo en UiPath.
Intermedio Flujo ReAct para Aprobación de Facturas ›

Diseña un flujo completo usando el framework ReAct para un agente que procesa facturas, toma decisiones y ejecuta acciones según reglas de negocio.

📋 Tu misión
  • Dibuja/describe el ciclo Thought → Action → Observation para al menos 3 iteraciones.
  • Define el punto HITL: ¿En qué condición el agente se detiene y pide aprobación?
  • Lista las tools necesarias: API de ERP, Slack, base de datos de proveedores.
  • Maneja el caso de error: ¿Qué pasa si la API del ERP falla?
✅ Criterios de éxito
  • El flujo ReAct tiene mínimo 3 ciclos completos documentados.
  • El punto HITL tiene un umbral numérico claro.
  • Existe un mecanismo de manejo de errores (retry + escalación).
Intermedio Análisis RPA vs Agentes: 5 Casos Reales ›

Dado un listado de procesos empresariales, decide si cada uno debe automatizarse con RPA puro, agente de IA, o una combinación híbrida. Justifica con criterios técnicos.

📋 Procesos a analizar
  • Generación diaria de 5,000 facturas con formato fijo desde SAP.
  • Clasificación y respuesta a emails de soporte al cliente.
  • Transferencia de datos entre dos sistemas con API documentada.
  • Análisis de contratos PDF para detectar cláusulas de riesgo.
  • Aprobación de solicitudes de vacaciones según política HR.
✅ Criterios de éxito
  • Cada caso tiene una recomendación clara con justificación de 2+ razones.
  • Identificas al menos 2 casos híbridos donde combinas ambas tecnologías.
  • Mencionas el riesgo principal de cada elección.
Intermedio Tool Registry para Agente de Atención al Cliente ›

Diseña el Tool Registry completo para un agente de atención al cliente de un e-commerce. Cada herramienta debe estar bien documentada para que el LLM la use correctamente.

📋 Tu misión
  • Define mínimo 6 herramientas con: nombre, descripción, parámetros y cuándo usarla.
  • Incluye herramientas para: consultar pedidos, actualizar estado, enviar email, escalar a humano, consultar políticas y procesar devoluciones.
  • Para cada tool, define qué NO debe hacer el agente con ella.
  • Establece el orden de preferencia si dos tools parecen válidas.
✅ Criterios de éxito
  • Cada herramienta tiene descripción en lenguaje natural (así la lee el LLM).
  • Los parámetros tienen tipos de datos y ejemplos.
  • Existe una tool de escalación con criterios claros.
Avanzado Arquitectura Multi-Agente: Onboarding de Empleados ›

Diseña un sistema multi-agente completo para automatizar el proceso de onboarding de nuevos empleados, coordinando múltiples departamentos y sistemas.

📋 Tu misión
  • Define el Agente Orquestador: cómo coordina los subagentes.
  • Define 4 Subagentes especializados: RRHH, IT, Facilities, Legal.
  • Diseña el flujo de delegación y hand-off entre agentes.
  • Identifica 3 puntos HITL en el proceso.
  • Define el mecanismo de rollback si algo falla.
✅ Criterios de éxito
  • Cada agente tiene un dominio claro y no se solapan.
  • El orquestador tiene criterios de delegación explícitos.
  • El rollback revierte acciones en orden inverso.
  • Se estima el tiempo total del proceso vs. proceso manual.
Avanzado Plan de Seguridad: Guardrails Completos ›

Diseña el plan de seguridad completo para un agente de automatización financiera, considerando las principales amenazas y mecanismos de protección.

📋 Tu misión
  • Simula 3 ataques de Prompt Injection y describe cómo el agente los detecta y rechaza.
  • Define el mecanismo de Grounding: ¿Qué fuentes son válidas y cuáles no?
  • Diseña el sistema de observabilidad: ¿Qué logs, métricas y alertas necesitas?
  • Establece la política de Rate Limiting por tipo de operación.
  • Define el proceso de Evals: cómo mides que el agente es confiable.
✅ Criterios de éxito
  • Los 3 ataques de Prompt Injection son casos realistas.
  • El sistema de observabilidad detecta alucinaciones automáticamente.
  • Las Evals incluyen casos de éxito y casos de fallo esperados.
Experto Diseño End-to-End: Agente de Ventas Autónomo ›

Proyecto integrador: diseña un agente de ventas completamente autónomo capaz de calificar leads, personalizar propuestas y hacer seguimiento, usando todos los conceptos del curso.

📋 Tu misión
  • Prompt agéntico completo con persona, objetivo, tools y restricciones.
  • Arquitectura de memoria: qué tipo usa en cada etapa del proceso.
  • Tool Registry: mínimo 8 herramientas documentadas.
  • Flujo ReAct: desde el primer contacto hasta el cierre.
  • Plan de HITL: cuándo escala a un vendedor humano.
  • Guardrails: qué información nunca puede compartir.
  • Métricas de éxito: KPIs para evaluar el agente en producción.
✅ Criterios de éxito
  • El diseño es coherente y no tiene contradicciones entre secciones.
  • Las métricas son medibles y tienen valores objetivo definidos.
  • Identificas al menos 3 riesgos y propones mitigación para cada uno.
  • El sistema escala correctamente cuando el agente no está seguro.
